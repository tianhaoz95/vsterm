Architecture

How it works

Two components, one WebSocket. No Microsoft servers in the path.

 Browser (vscode.dev)
 ┌──────────────────────────────────────────────┐
 │  vsterm extension                            │
 │  • xterm.js terminal rendered in a Webview   │
 │  • Extension host opens WebSocket to daemon  │
 │  • Keystrokes → daemon, PTY output → xterm.js│
 └──────────────────────┬───────────────────────┘
                        │  ws://127.0.0.1:7007
                        ▼
 Local machine
 ┌──────────────────────────────────────────────┐
 │  vsterm daemon  (~8 MB Go binary)           │
 │  • Bound to 127.0.0.1 — local only          │
 │  • Spawns PTY shells via creack/pty          │
 │  • Multiplexes N terminals per browser tab   │
 └──────────────────────────────────────────────┘
⚡

Zero latency

Traffic never leaves your machine. The WebSocket is a loopback connection — as fast as a local socket can be.

🔒

Secure by default

Daemon binds 127.0.0.1 only. CORS is restricted to vscode.dev. No tokens, no TLS needed on localhost.

🖥️

Full terminal emulation

Powered by xterm.js — the same renderer VS Code itself uses. Full VT100/ANSI, colors, resize, scrollback.

📑

Multiple terminals

Tab bar with + to open more terminals and × to close them. Each tab is an independent PTY session.

🔄

Auto-reconnect

If the browser tab is closed and reopened, the extension reconnects automatically with exponential backoff.

💾

Session persistence via tmux

Run tmux in your terminal. After a reconnect, tmux attach restores your session exactly as you left it.

Getting started

Up and running in 2 steps

No configuration files, no accounts, no ports to forward.

1

Install the daemon

Run npm install -g vsterm in your terminal. The daemon starts immediately and registers as a login service — it will be running every time you boot.

One-shot alternative: npx vsterm starts it in the foreground without installing.

2

Open a terminal in vscode.dev

Install the vsterm extension from the VS Code Marketplace, then run vsterm: Open Local Terminal from the command palette. That's it.
